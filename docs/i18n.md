# Internationalization (i18n)

> **Status: IMPLEMENTED** — Language routing, auto-detection, default language, and language management API are all in place.

## Concepts

- **Site languages** — managed by the admin through the admin panel. Stored in the `languages` table.
- **Admin UI languages** — fixed set: `en` and `ru`. Toggled by the admin user inside the admin panel.

These are two separate i18n concerns.

## URL Structure (Public Client)

All pages are served under a language prefix:

```
/{lang}/{slug}

/en/about
/ru/about
/et/kontakt
```

The `{lang}` segment must be one of the **active** languages configured in the admin panel.

## Root URL Redirect

When a visitor opens `/`, the public client:

1. Reads `Accept-Language` header from the browser
2. Matches it against the list of active languages from the API
3. Redirects to `/{matched-lang}/` (homepage slug)
4. If no match → redirects to the **default language** — the language marked `isDefault` in the admin panel, falling back to the first active language, then `"en"`

Implemented in `apps/web/src/pages/index.astro` and `apps/web/src/lib/resolveDefaultLang.ts`.

## Path Without Language Prefix

If a visitor opens a path without a language segment (e.g. `/about`), it is caught by `apps/web/src/pages/[...slug].astro` and redirected to `/{resolvedLang}/about` using the same language resolution logic.

Similarly, if the first segment of a URL is not a known active language code, `apps/web/src/pages/[lang]/[...slug].astro` detects this, reconstructs the full slug, and redirects to `/{resolvedLang}/{fullSlug}`.

## Language Switching

The public client header displays a language switcher. It shows **only active languages** fetched from the API at build time.

Clicking a language navigates to the same slug in the selected language:
```
/en/about  →  /ru/about
```

## Admin Panel UI Language

Not yet implemented — admin UI is English-only for now.

## Default Language

Each language row has an `isDefault` boolean. Only one language can be default at a time — updating one automatically clears the flag on all others (enforced in `apps/api/src/db/repository/languages.ts`).

The default language can be set in the admin panel under **Languages** → "Set default" button.

## Language Management API

| Method | Endpoint | Description |
|---|---|---|
| `GET` | `/languages` | List all languages |
| `POST` | `/languages` | Add a new language |
| `PATCH` | `/languages/:id` | Update label, active status, or isDefault |
| `DELETE` | `/languages/:id` | Remove a language |
