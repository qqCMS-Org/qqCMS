# Public Web Client (`apps/web`)

> **Status: IMPLEMENTED** вЂ” Phase 7 (Steps 1вЂ“6) is complete. See `docs/TODO.md` for the full checklist.

## Overview

`apps/web` is the public-facing website generated by qqCMS. It is built with Astro in **static** mode (Astro v5+):

- Content pages (`/[lang]/[...slug]`) are **prerendered at build time** (SSG) вЂ” every published page Г— every active language becomes a static HTML file.
- The root `/`, `/api/revalidate`, and `404.astro` are **SSR** вЂ” they need request-time data (`Accept-Language` header, deploy hook) or cannot be prerendered.

Architecture mirrors `apps/admin` exactly: same FSD layers, same path aliases, same Eden Treaty client pattern, same Tailwind + DaisyUI setup.

## Stack

| Component | Technology |
|---|---|
| Framework | Astro (hybrid mode) |
| UI islands | Preact (same as admin) |
| Shared primitives + TipTap renderer | `@repo/ui` |
| Styling | Tailwind CSS + DaisyUI (same theme as admin) |
| API client | Eden Treaty (`@repo/server/client`) |

## Content rendering lives in `packages/ui`

TipTap node components are **Preact TSX** in `packages/ui/src/tiptap/`. Preact components imported in Astro pages without a `client:*` directive render as static HTML вЂ” zero JS shipped to the browser.

This means the same components can later be used in `apps/admin` for a live page preview, without any duplication.

## Adding a Custom Content Block

To add a custom TipTap node (e.g., a "hero" full-width banner):

**1. Create the renderer in `packages/ui`**

```tsx
// packages/ui/src/tiptap/nodes/HeroNode.tsx
import type { TipTapNode } from "../types"

interface Props {
  node: TipTapNode
}

export const HeroNode = ({ node }: Props) => (
  <section class="hero">
    <img src={node.attrs?.imageUrl} alt={node.attrs?.title} />
    <h2>{node.attrs?.title}</h2>
    {node.attrs?.buttonText && (
      <a href={node.attrs.buttonHref}>{node.attrs.buttonText}</a>
    )}
  </section>
)
```

**2. Register it**

```ts
// packages/ui/src/tiptap/registry.ts вЂ” add one line
import { HeroNode } from "./nodes/HeroNode"

export const defaultRegistry: NodeRegistry = {
  // ... existing entries ...
  hero: HeroNode,
}
```

**3. Add the TipTap extension in `apps/admin`** (separate step вЂ” see TipTap extension docs).

No changes to `TipTapRenderer.tsx` or any page file in either app.

## Environment Variables

| Variable | Description | Required |
|---|---|---|
| `PUBLIC_API_URL` | URL of the API server | Yes |
| `REVALIDATE_SECRET` | Shared secret for the `/api/revalidate` webhook | Yes |
| `DEPLOY_HOOK_URL` | Render.com (or other) deploy hook to trigger rebuild | Production only |

## Rebuild Flow

1. Admin saves content в†’ API calls `POST {PUBLIC_CLIENT_URL}/api/revalidate`
2. `revalidate.ts` verifies `x-revalidate-secret` with `crypto.timingSafeEqual`
3. Valid в†’ calls `DEPLOY_HOOK_URL`; absent in dev в†’ returns `{ ok: true }`
4. New build fetches fresh data в†’ regenerates all static HTML
